## 이번 세션에 만든 것 — 트래커 (/tracker)

### 구현 완료

**파일**
- `app/tracker/page.tsx` — 트래커 페이지
- `components/TrackerClient.tsx` — 트래커 전체 클라이언트 컴포넌트

**기능**

- 일자별 보기 (← 오늘 →) + 과거 날짜에도 기록 가능

- 인사이트 패널 (마지막 수유 후 배고픔 예측, 수면 중 경과 시간, 기저귀 4시간 이상 알림)
- 전체 데이터 localStorage 저장

## 다음에 할 것 (우선순위 순)

### 1순위 — 데이터 안전성 (중요)

현재 localStorage만 사용 → 브라우저 바꾸면 기록 날아감 (카카오 인앱 → 사파리 등)

**옵션 A: JSON 내보내기/가져오기** (빠름, 1~2시간)
- 트래커 하단에 "데이터 내보내기" 버튼 → JSON 파일 다운로드
- "가져오기" 버튼 → JSON 파일 업로드로 복원
- `TrackerClient.tsx`에 export/import 함수 추가

**옵션 B: Supabase 익명 auth + 클라우드 동기화** (중간, 반나절)
- 첫 접속 시 익명 UUID 생성 → Supabase `tracker_entries` 테이블에 저장
- 같은 UUID로 다시 접속하면 데이터 복원
- UUID를 localStorage에 저장 (브라우저 바꾸면 여전히 끊김, 하지만 의도적 공유 가능)

**옵션 C: 카카오/구글 로그인** (장기)
- Supabase Auth 연동
- 완전한 크로스 디바이스 동기화

→ **당장은 A부터** 시작 권장

## 현재 알려진 이슈

- localStorage 격리 문제 (카카오 인앱 ↔ 사파리 데이터 공유 안 됨)
- PWA 푸시 알림 미구현 (iOS Safari PWA 푸시는 16.4+ 필요, 불안정)
  - 현재: 인사이트 패널로 대체 (앱 열었을 때 배고픔 예측 표시)
  - 향후: 네이티브 앱(Expo/React Native) 전환 시 구현 가능
